# 前记

这几天突然对服务器学习感了兴趣，主要还是来源于之前对Linux的学习吧哈哈哈哈。然后就去Linux上白嫖了两个月的服务器，熟悉了一下CentOS的基本操作，便想着在服务器上部署一个网站，查来查去，就选择采用vuepress构建一个博客。

但是完成这么几项任务也耗费了很长（大概一天半的时间），可能是我比较菜😂，也踩了不少坑，姑且写一下，供自己查看。

# 安装

  1. 安装node.js框架

    直接去node.js官网 (opens new window)下载就好

    Windows

    直接安装应用程序就好。

    安装完毕可以在命令行检查，输入以下指令

    node -v

    若有版本号出现，则安装成功

    Linux

    对于Linux是一个压缩包。

    先解压文件

    tar zxvf node-v0.10.24.tar.gz

    编译安装

    cd node-v0.10.24
./configure --prefix=/usr/local/node/0.10.24
make
make install

    配置环境变量

    vim /etc/profile

    在 export PATH USER LOGNAME MAIL HOSTNAME HISTSIZE HISTCONTROL 一行的上面添加如下内容:

    #set for nodejs
export NODE_HOME=/usr/local/node/0.10.24
export PATH=$NODE_HOME/bin:$PATH

    :wq保存并退出，编译/etc/profile 使配置生效

    source /etc/profile

    验证是否安装配置成功

    node -v

    输出 v0.10.24 表示配置成功

    npm模块安装路径

    /usr/local/node/0.10.24/lib/node_modules/

  2. 安装yarn

    npm install -g yarn

  3. 安装vuepress

    在一个地方创建一个文件夹，用包管理器进行初始化

    yarn init # npm init

    将 VuePress 安装为本地依赖

    yarn add -D vuepress # npm install -D vuepress

# 初步上手

安装了以上所需包，就可以进行初步上手了，在构造一个静态网页之前，还要进行以下工作。

# package配置

在 package.json 中添加一些 scripts

{
  "scripts": {
    "dev": "vuepress dev docs",
    "build": "vuepress build docs"
  }
}

推荐做此项工作，这是把命令vuepress dev docs编译成一个脚本，这样使用包管理器npm或yarn使项目运行时，只需输入

yarn dev # npm run dev

# 添加主页

在目录下创建docs文件夹，在其中添加README.md文件，可以输入# Hello VuePress

输入上文中运行指定，便可在 http://localhost:8080 (opens new window) (opens new window)启动一个热重载的开发服务器。

# 目录结构

VuePress 遵循 “约定优于配置” 的原则，推荐的目录结构如下：

.
├── docs
│   ├── .vuepress (可选的)
│   │   ├── components (可选的)
│   │   ├── theme (可选的)
│   │   │   └── Layout.vue
│   │   ├── public (可选的)
│   │   ├── styles (可选的)
│   │   │   ├── index.styl
│   │   │   └── palette.styl
│   │   ├── templates (可选的, 谨慎配置)
│   │   │   ├── dev.html
│   │   │   └── ssr.html
│   │   ├── config.js (可选的)
│   │   └── enhanceApp.js (可选的)
│   │ 
│   ├── README.md
│   ├── guide
│   │   └── README.md
│   └── config.md
│ 
└── package.json
  • docs/.vuepress: 用于存放全局的配置、组件、静态资源等。
  • docs/.vuepress/components: 该目录中的 Vue 组件将会被自动注册为全局组件。
  • docs/.vuepress/theme: 用于存放本地主题。
  • docs/.vuepress/styles: 用于存放样式相关的文件。
  • docs/.vuepress/styles/index.styl: 将会被自动应用的全局样式文件，会生成在最终的 CSS 文件结尾，具有比默认样式更高的优先级。
  • docs/.vuepress/styles/palette.styl: 用于重写默认颜色常量，或者设置新的 stylus 颜色常量。
  • docs/.vuepress/public: 静态资源目录。
  • docs/.vuepress/templates: 存储 HTML 模板文件。
  • docs/.vuepress/templates/dev.html: 用于开发环境的 HTML 模板文件。
  • docs/.vuepress/templates/ssr.html: 构建时基于 Vue SSR 的 HTML 模板文件。
  • docs/.vuepress/config.js: 配置文件的入口文件，也可以是 YML 或 toml。
  • docs/.vuepress/enhanceApp.js: 客户端应用的增强。

docs文件夹在改名时，需对package.json文件script中对应名字更改。

只定位到文件夹时，打开的网页为文件夹名.md或者文件夹内的README.md，切忌不能同时出现！！

# config.js 配置

如下

module.exports = {
  title: 'Hello VuePress',
  description: 'Just playing around'
}
  • title为网页名称
  • description为网页介绍

以下为官方默认主题常用配置

导航栏链接

你可以通过 themeConfig.nav 增加一些导航栏链接:

// .vuepress/config.js
module.exports = {
  themeConfig: {
    nav: [
      { text: 'Home', link: '/' },
      { text: 'Guide', link: '/guide/' },
      { text: 'External', link: 'https://google.com' },
      {
        text: 'Languages',
        ariaLabel: 'Language Menu',
        items: [
          { text: 'Chinese', link: '/language/chinese/' },
          { text: 'Japanese', link: '/language/japanese/' }
        ]
      }
    ]
  }
}
  • item可以提供一个下拉链接

其余配置可以见官方文档 (opens new window)和官方默认主题配置文档 (opens new window)

# Front Matter

front matter 必须是 markdown 文件中的第一部分，并且必须采用在三点划线之间书写的有效的 YAML。规定了md文件的基本配置，常用可见下：

---
{
  "title": "Blogging Like a Hacker", # 本页面的标题
  "lang": "en-US", # 本页面的语言
  "description": , # 本页面的描述
  "navbar": false, # 禁用导航栏
  "sidebar": false, # 禁用标题栏
}
---

其余可见官方文档 (opens new window)

# VuePress 主题

利用主题可以方便地建立一些网站，比如blog，主题也加入了不少功能便于调用。为了搭建一个博客，我尝试了以下主题：

  • 官方博客主题 (opens new window)

    这个主题增加了文章分类功能，个人觉得这个文章分类比较合理，但是功能较少，主页不好看，故没有选择。

  • vuepress-theme-hope (opens new window)

    老实说这个主题比较合我意，但是配置时总是出错🙃，不知道是什么原因

  • vuepress-theme-reco (opens new window)

    最后选用的这个主题，但是这个主题的文章分类是通过front matter中的一个categories参数确定的，不是官方博客主题中的按照文件夹分类，可能这种分类方式比较简便，毕竟只需要改变每个文档中的front matter，但是个人的逻辑还是喜欢按照文件夹。

以下记录一下自己常用的配置，具体可去官方文档中查询。

# 服务器部署

# 迁移

可以先在服务器上建立环境，再将本地上文件利用Xftp迁移过去，或者使用github或gitee仓库进行代码更新。

建立环境

mkdir my_blog
cd my_blog
yarn add -D vuepress
yarn add vuepress-theme-reco
  • 按道理只用add主题就够了，自身就包含了vuepress，但是只add主题启动后会报错，原因未知。

# 运行

运行前最好build一下，生成静态网页文件（好处未知，正在查阅资料）

yarn build
yarn dev

然后在http://公网IP：8080中便可建立博客。

注意

使用ssh连接服务器时，如果断开ssh连接，网页运行会自动停止，因此需要使用以下指令

nohup npm run dev > test.log & exit
  • 按道理yarn dev也可以，但是用yarn会报错，原因未知。
  • test.log是指令输出的日志。
  • nohup英文全称 no hang up（不挂起），后加&指让命令在后台执行，终端退出后命令仍旧执行。exit退出ssh连接。

# 终止运行

终止运行可以用以下指令

ps -aux

然后找到网页运行的程序，输入

kill -9 进程号PID

# 目录

.
├─ README.md
├─ daily
│  ├─ README.md
│  └─ ...
├─ note
|  ├─ README.md
│  └─ ...
├─ private
|  ├─ README.md
│  └─ ...
├─ interview
|  ├─ README.md
│  └─ ...
├─ pic
│  └─ ...
└─ .vuepress
   ├─ config.js
   ├─ public/
   └─ dist/

  • /daily/为生活日常文件夹

  • /note/为学习日志文件夹

  • /private/为个人感悟随笔文件夹

  • /interview/为个人介绍文件夹，即在导航栏中的About项

  • /pic/为文中的照片存储，期望在里面也进行文件分类

  • /.vuepress/config.js为项目配置文件

    /.vuepress/public为推送照片存储，一般为头像

    /.vuepress/dist为静态网页文件

# 更新文章

  • 如果更新的为生活日常或感悟随笔，直接在front matter中加入

    categories:
 - 生活日记
 - 感悟随笔
tag:
 - 随便写（不写也行，建议不写）

  • 如果更新学习日记，如果想引入导航栏中的Note除去改变categories，还需对侧边栏进行修改。

    // .vuepress/config.js

...
sidebar: {
      '/note/': [
        '',     
        'TODO',  
		'加入', //这里写路径及文件名
        {
          title: '二级笔记',
          collapsable: true,
          children: [
            '1-1',
            '1-2',
            '1-3'
          ]
        },
      ]
}
...
  • 更新个人介绍也是一样，需要修改config.js中侧边栏中对应内容。
